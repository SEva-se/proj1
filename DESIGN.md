# DESIGN.md — дизайн-система INFO CLINIC

Единый источник правды по визуалу. Любой новый элемент должен собираться из этих
токенов и принципов. Цель — **«частная диагностическая клиника премиум-класса»**:
тёмный кабинет, точные приборы, живой пульс. Не неоновый кибер-панк, не плоский
корпоратив, не дефолтный ИИ-градиент.

## 1. Принципы

1. **Метафора прежде декора.** Каждый новый элемент отвечает на вопрос «как это
   выглядело бы в высокотехнологичной клинике?»: приборные панели, ЭКГ, карты
   пациента, печати/пломбы, рукописные пометки врача, мониторинг.
2. **Спокойная тьма + точечный свет.** 95% экрана — глубокий тёмный фон, акцент
   `vital`-зелёным даётся скупо (живой сигнал на фоне покоя).
3. **Тишина между блоками.** Воздух, тонкие разделители-линии, мягкие radial-glow.
   Плотность информации высокая, но не шумная.
4. **Антишаблон.** Никаких стоковых градиентных «blob», карточек с одинаковыми
   эмодзи, центрированного «всё по центру». Ассиметрия, повороты карточек, mono-
   подписи как серийные номера, «человеческие» рукописные акценты.

## 2. Цветовые токены (`tokens.css`)

Брать из исходника без изменений палитры — она сильная.

```css
:root{
  /* поверхности */
  --ink:#08090B;        /* фон страницы */
  --surface:#0D0F13;
  --surface-2:#13161C;
  --card:#15181E;

  /* текст */
  --bone:#ECEAE4;       /* основной текст */
  --mute:#8B9099;       /* вторичный */
  --mute-2:#5A6068;     /* третичный / mono-подписи */

  /* сигнальный зелёный — «жизненные показатели» */
  --vital:#3FDD9A;
  --vital-soft:rgba(63,221,154,.09);
  --vital-line:rgba(63,221,154,.32);

  /* акценты-предупреждения */
  --gold:#C9A24B;  --gold-soft:rgba(201,162,75,.10); --gold-line:rgba(201,162,75,.32);
  --amber:#D7944C; --amber-soft:rgba(215,148,76,.08); --amber-line:rgba(215,148,76,.28);

  /* линии */
  --line:rgba(236,234,228,.09);
  --line-2:rgba(236,234,228,.055);

  /* геометрия / тайминги */
  --r:18px; --r-sm:12px; --maxw:1180px;
  --ease:cubic-bezier(.2,.7,.3,1);
}
```

Семантика цвета: `vital` = «здоровье/решение/CTA», `amber` = «было/симптом»,
`gold` = «внимание/премиум-метка», `#E0644A` = «ошибка» (красная точка монитора).

## 3. Типографика

| Роль | Шрифт | Начертания | Где |
|---|---|---|---|
| Заголовки + текст | **Montserrat** | 400 / 500 / 600 / 700 / 800 | всё читаемое |
| Каллиграфический акцент | **Caveat** (есть кириллица) | 500 / 700 | «почерк врача»: 1–3 слова на блок |
| Технические лейблы | **JetBrains Mono** | 400 / 500 | eyebrow, статусы, серийники, цифры приборов |

- **Unbounded удалён.** Нигде не использовать.
- **Caveat — строго дозированно.** Это «пометка от руки в карте пациента»: одно
  выделенное слово в заголовке, подпись под печатью, акцент в закрывающем абзаце.
  Не заголовки целиком, не абзацы, не кнопки. Цвет — `vital` или `gold`, лёгкий
  наклон уже встроен в шрифт. Никогда не для слов, которые меняют смысл текста
  (текст неприкосновенен — только визуальное выделение уже существующих слов).
- Заголовки: `font-weight:700–800; letter-spacing:-.02em; line-height:1.1`.
- Текст: `font-weight:400; line-height:1.6; color:var(--mute)` для вторичного.

Шкала (clamp, fluid):
```
h1   clamp(32px, 5vw, 60px)
h2   clamp(26px, 4vw, 46px)
h3   clamp(19px, 2.4vw, 26px)
lead clamp(16px, 1.8vw, 19px)
body 16px
mono 10–11px / letter-spacing .14–.2em / uppercase
```

Подключение (Google Fonts, сабсет latin+cyrillic):
`Montserrat:wght@400;500;600;700;800` · `Caveat:wght@500;700` ·
`JetBrains+Mono:wght@400;500`.

## 4. Движение (анимации)

- Базовый ease — `--ease` (`cubic-bezier(.2,.7,.3,1)`), длительности 250–850ms.
- **Reveal-on-scroll**: `.rv{opacity:0;transform:translateY(22px)}` → `.in`,
  через `IntersectionObserver` (как в исходнике, сохранить). Каскад `.d1/.d2/.d3`.
- **Живые сигнатуры клиники** (ключевая «фишка», масштабировать):
  - ЭКГ-линия `stroke-dashoffset` (`@keyframes draw`) — пульс в hero и разделителях.
  - `ping` — пульсация вокруг play/точек статуса.
  - `blink` — REC-индикатор, `pulse` — «онлайн».
  - `scroll` — бегущая лента скриншотов (пауза по hover).
- **Hover карточек**: `translateY(-3px)` + смена `border-color` на `--vital-line`.
- **Курсор-параллакс** фона hero — лёгкое смещение glow за мышью (≤8px), только desktop.
- 🔴 **Обязательно** уважать `@media (prefers-reduced-motion:reduce)` — гасить всё
  (в исходнике уже реализовано в каждом блоке, не потерять при рефакторе).

## 5. Фон («интересный задний фон»)

Слои фона (фиксированный, `pointer-events:none`, `z-index:0`):
1. Базовая тьма `--ink`.
2. **Сетка-«миллиметровка»** как на ЭКГ-ленте/медбланке:
   `linear-gradient` 54×54px, `mask` radial — проявляется только у верха блока.
3. **Radial-glow** `vital` 5–6% в правом-верхнем углу — «свет прибора».
4. (Новое) Тонкая «изолиния пульса» сверху страницы, медленно ползущая.
5. (Новое, опц.) Едва заметный шум/grain `opacity:.02` — убирает «пластиковость»
   и делает картинку менее «ИИ-гладкой».
Не добавлять анимированные цветные blob и full-screen видеофоны — это и есть
признак шаблона.

## 6. Иконки и глифы («побольше иконок и эмодзи»)

Заказчик хочет богатую, «живую» иконографику (как в референсах). Используем её
**щедро**, но в едином премиальном языке — иконок много, но они одностильные.

- Базовый стиль: **линейные SVG**, `stroke-width:1.6–2.4`, `fill:none`,
  `stroke-linecap/linejoin:round`, размер 16–28px, цвет через `currentColor`/`--vital`.
- Иконки часто сидят в **«капсуле»**: кружок/квадрат `--r-sm` с фоном `--vital-soft`
  (или `--gold-soft`) и тонкой рамкой `--vital-line` — как в украинском и Monex
  референсах (цветная иконка в светлой плашке). Это даёт «побольше иконок» без шума.
- Тематический набор (расширять сюда): пульс/ЭКГ, стетоскоп, сердце, шприц/пробирка,
  рентген, капсула, крест/плюс, микроскоп, карта пациента, печать/пломба, чек-галка,
  стрелка-переход, термометр, monitor/waveform, щит, молния, часы.
- **Декоративные глифы-«искры»** (из wuizt/Monex): маленькие 4-лучевые звёздочки/
  плюсики `--vital`/`--gold` с лёгким glow, разбросанные у заголовков и карточек как
  «блёстки данных». Используем вместо стоковых блобов.
- **Эмодзи — дозированно и в тон.** Разрешены как точечный акцент энергии у CTA/
  бейджей (🔥 ⚡ ✦ ⚕ ✓ →), 1 шт. на блок максимум, не в основном тексте абзацев,
  не в заголовках h1/h2. Если эмодзи спорит с премиальностью — заменить линейной
  иконкой в капсуле. Никогда не менять/не добавлять смысловой текст эмодзи-значками.
- Источник: рисовать вручную или брать в одном стиле (Lucide / Phosphor) и приводить
  к токенам. Хранить как `ui/icons/*.astro`, красить через `currentColor`.

## 7. Компоненты (паттерны из исходника — сохранить и привести к единому виду)

- **Button** `.btn` — пилюля `border-radius:100px`, фон `--vital`, текст `#04140C`,
  hover `translateY(-2px)+glow`. 🔴 На лендинге всегда ведёт на анкету. Текст не менять.
- **Eyebrow** — mono-надзаголовок с пульсирующей точкой `vital`.
- **Monitor / приборная панель** — рамка с тремя «светофорными» точками
  (`#E0644A / gold / vital`) и mono-лейблом справа. Обёртка для скринов/видео.
- **Карта пациента** (`.pc`) — карточка кейса: badge, статус-точка, симптом → диагноз.
- **Сравнение** (`.cmp`) — две колонки «было / стало», правая подсвечена `vital-soft`.
- **Печать/пломба** (`.seal`) — круг с галкой, `pop`-анимация (финал формы).
- **Frame-уголки** (`.corner`) — 4 уголка `vital` вокруг hero-блока.
- Новые в том же языке: «лента ЭКГ-разделитель» между секциями, «карточка-рецепт»
  с рукописным Caveat-акцентом, «бейдж серийного номера» (mono).

## 8. Адаптив
- Контент-ширина `--maxw` (1080–1180px по блоку), padding контейнера 24px (20px ≤560).
- Брейкпоинты из исходника: 560 / 760 / 880 / 900 / 980 / 1000px — не плодить новые.
- Кнопки на мобильном — `width:100%`.
- Тач: убрать hover-параллакс, не полагаться на hover для контента.

## 9. Стоп-лист (чтобы не выглядело «как очередной ИИ-сайт»)
- ❌ Фиолетово-синий градиент по умолчанию (палитра у нас своя — тьма + vital/gold).
- ❌ Эмодзи в основном тексте, в h1/h2, гирляндой по 5 штук подряд.
- ❌ Стоковые иллюстрации «3D-человечки», цветной glassmorphism везде.
- ❌ Всё по центру, одинаковые отступы, единый размер шрифта.
- ❌ Caveat на больших объёмах текста / в кнопках.
- ✅ Ассиметрия, повороты карт пациента, приборные детали, рукописные пометки,
  серийные номера, живой пульс, скупой акцентный свет, парящие карточки-пруфы.

## 10. Референсы и заимствованные паттерны

От заказчика: wuizt (тёмный фиолетовый AI-SaaS), украинский лендинг запуска курса,
Monex (светлый зелёный финтех). **Берём только композицию и приёмы — каркас (блоки
`#blk1..12` + медицинская метафора) и палитру (тьма + vital + gold) НЕ меняем.**
Цель позиционирования — премиальный сайт по продаже **менторства/диагностики**.

| Приём из референса | Как адаптируем под INFO CLINIC |
|---|---|
| Парящие UI-карточки вокруг мокапа (wuizt, Monex) | «Парящие карточки жизненных показателей» (выручка, конверсия, пульс проекта) вокруг центрального «монитора-прибора» в hero. Лёгкий float + параллакс. |
| Соединительные линии-ноды между карточками (wuizt) | Тонкие `--vital-line` коннекторы с узлами-точками — «схема диагностики / ЭКГ-разводка». |
| Декоративные «искры»-звёздочки + glow (wuizt, Monex) | Глифы-искры `vital`/`gold` у заголовков (см. §6). |
| Кластер аватаров + бейдж «230+» (украинский) | Кластер аватаров пациентов + mono-бейдж «100+ обследований» (текст уже есть в исходнике — не выдумывать новый). |
| Ряд мини-карточек с цветными иконками (украинский) | Иконка-в-капсуле + короткая подпись (уже есть `.gain`, `.pc` — усилить иконками §6). |
| Сплит-заголовок двумя цветами (украинский) | Уже делаем через `.hl` (vital). Доп. акцент — слово в Caveat (§3). |
| Кнопка-пилюля с круглой стрелкой ↗/→ (Monex, wuizt) | Добавить иконку-стрелку в `.btn` (как в `Get started ↗`). Текст кнопок не менять. |
| Мягкий радиальный glow-ореол за мокапом (Monex) | Уже есть в фоне (§5) — усилить под центральным прибором hero. |
