# CLAUDE.md — INFO CLINIC

Гайд для любой будущей сессии Claude по этому проекту. Читай целиком перед изменениями.

## Что это за проект

Лендинг + анкета продающей услуги **«INFO CLINIC · Диагностика проекта»** —
бизнес-диагностика экспертов и онлайн-школ (автор — Рустам Михайлов,
[@mihaylov.rustam](https://www.instagram.com/mihaylov.rustam)).

Центральная идея — **медицинская метафора**: бизнес = пациент, маркетинг =
диагностика, отчёт = диагноз и план лечения. Вся айдентика (ЭКГ, «карта
пациента», «vital»-зелёный, золото, mono-лейблы как на медприборе) держится на
этой метафоре. **Метафору не размывать — это то, что отличает сайт от шаблонного
ИИ-лендинга.**

Позиционирование — **премиальный сайт по продаже менторства/диагностики**. Уровень
исполнения ориентируем на присланные референсы (wuizt, Monex, украинский лендинг
запуска): парящие UI-карточки, мягкое свечение, богатая иконография, сильный
соц-пруф. **Берём приёмы, но каркас и палитру не меняем** — детали в
[DESIGN.md](DESIGN.md) §10.

## 🔴 ГЛАВНОЕ ПРАВИЛО — текст неприкосновенен

**Запрещено менять любой видимый текст и копирайт.** Заголовки, абзацы, отзывы,
названия кнопок, подписи, «INFO CLINIC · ЧАСТНАЯ ДИАГНОСТИКА», `★★★★★ 5.0 · 100+
обследований`, ссылка на инстаграм — всё остаётся дословно как в исходниках.

Разрешено менять: вёрстку, CSS, разметку, шрифты, анимации, фон, иконки, добавлять
новые декоративные элементы (без нового текста-сообщения), структуру файлов,
порядок DOM — **но не сами слова**. Если кажется, что текст нужно поправить —
не трогать, спросить владельца.

Перед коммитом — сверять видимый текст с исходниками:
`Сайт новый.html` и `страница заявки.html` (лежат в корне как референс, не удалять).

## Технический стек

- **Astro + Vite** — статическая сборка, ноль лишнего JS, авто-оптимизация картинок
  (`astro:assets` → WebP/AVIF), сабсеттинг шрифтов, минификация.
- Анимации и интерактив — **vanilla JS** в островках/`<script>`, без тяжёлых
  библиотек. Уважать `prefers-reduced-motion` (в исходниках уже есть — сохранить).
- Деплой — статика (Netlify / Vercel / Cloudflare Pages).

### Целевая структура
```
src/
  pages/
    index.astro            # лендинг (бывш. blk1..blk12)
    zayavka.astro          # анкета
  components/
    blocks/Blk1Hero.astro ... Blk12Cta.astro
    ui/ (Button, Eyebrow, Monitor, Badge, IconList...)
    fx/ (Background.astro, ECG.astro, Reveal.astro)
  styles/
    tokens.css             # единственный источник дизайн-токенов (см. DESIGN.md)
    base.css               # сброс, типографика, утилиты
  scripts/ (reveal.ts, form.ts)
  assets/img/              # вынутые из base64 картинки (WebP)
public/
```

## Что переделать (приоритеты)

1. **Вынести 41 base64-картинку** в `src/assets/img/`, отдать через `astro:assets`
   (WebP/AVIF, `loading="lazy"`, нужные размеры). Это главный выигрыш скорости.
2. **Кнопки лендинга → форму.** Сейчас CTA ведут на `#cta`/`#`. Все `.btn`
   на лендинге должны вести на страницу анкеты (`/zayavka`). Текст кнопок не менять.
3. **Починить отправку формы.** Сейчас в коде `// here you would POST answers`,
   а TG-ссылка — заглушка `https://t.me/your_bot_link`. Подключить реальный
   приёмник заявок (бот/CRM/webhook — уточнить у владельца) и реальную ссылку.
4. **Дедуп CSS.** У каждого блока свой `:root` с одинаковыми токенами (≈12 копий)
   → один `tokens.css`.
5. **Типографика:** Unbounded убрать. Montserrat — весь текст и заголовки; Caveat —
   каллиграфические акценты («почерк врача»); JetBrains Mono — тех-лейблы. См. DESIGN.md.
6. **Новые элементы дизайна и иконки** — в рамках метафоры (см. DESIGN.md).
